    Step 1: O1 查询理解                      │
    ┌───────────────────────────────────────────────────────────────┐
    │  • 意图分类：definitional / comparative / genealogical /               │
    │    critical / analytical / practical / casual                            │
    │  • 领域检测：哲学/社会学/政治学/文化研究/精神分析/文学理论/ACGN/历史/经济学  │
    │  • 复杂度评估：0.0 → 1.0 → 决定 Token 预算和策略                      │
    │  • 策略选择：debate / chain_of_thought / counterfactual / self_consistency │
    └───────────────────────────────────────────────────────────────┘
                                        │
    Step 2: O2 记忆检索                      │
    ┌───────────────────────────────────────────────────────────────┐
    │  📕 BM25 检索概念（字符n-gram，无需embedding）→ 命中 top 6 概念        │
    │  📚 书目检索（关键词匹配 impact_note / 标题 / 作者）→ 命中 top 3       │
    │  🗺️  Subgraph遍历（2跳架构）→ 揭示概念间的关系边（CONTRADICTS/EXTENDS等）      │
    │                                                                             │
    │  此时会检测："权力"是否有福柯版本 vs 韦伯版本？若有，标注概念役体化差异。        │
    └───────────────────────────────────────────────────────────────┘
                                        │
    Step 3: O3 无意识分析 ←─ 你看不到这一步──────────────────────────
    ┌───────────────────────────────────────────────────────────────┐
    │  🎭 内部隐式分析，结果不被输出                                  │
    │  • 张力检测：查询是否假定了统一性而实际上存在多重解释                    │
    │  • 矛盾检测：检查图谱中是否有CONTRADICTS边连接检索到的概念                │
    │  • 悖论检测："怎么办"类问题暗含求解决方案，但结构性问题无个人解答         │
    │                                                                             │
    │  关键原则：这些张力 FELT 于回复中，不被 STATED。                         │
    │  不会写"这里有个矛盾"，而是让措辞、引文选择本身承载张力。              │
    └───────────────────────────────────────────────────────────────┘
                                        │
    Step 4: O4 上下文组装                      │
    ┌───────────────────────────────────────────────────────────────┐
    │  组装一个系统提示（system prompt）给 LLM，包含：                       │
    │                                                                             │
    │  ① 硬约束清单（无人称/引文锚定/矛盾揭示/无结论收束等）                     │
    │  ② 检索到的概念（根据查询类型展开不同层级）                                 │
    │     • 默认: tier1（5-10字概念名+核心定义）                                   │
    │     • 比较/谱系查询: tier2（20-30字完整定义）                              │
    │     • 用户要求"详细": tier3（完整引文+定义）                                │
    │  ③ 概念关系边（CONTRADICTS/EXTENDS/APPLIES_TO）                              │
    │  ④ 无意识层指引（内部张力信号）                                            │
    │  ⑤ 相关书目冲击笔记（选择性加载）                                          │
    │                                                                             │
    │  Token总量: ~5.3K ← 旧系统的 65%                                          │
    └───────────────────────────────────────────────────────────────┘
                                        │
    Step 5: O5 LLM生成 + 有界自我反思              │
    ┌───────────────────────────────────────────────────────────────┐
    │  ① LLM生成回复（基于组装的上下文）                                      │
    │                                                                             │
    │  ② 自我反思（无LLM调用，零token消耗）：                                    │
    │     • 是否回答了问题？（字符覆盖率检查）                                  │
    │     • 是否引用了著作/思想家？（《》或"指出"检测）                         │
    │     • 是否有张力/矛盾？（"然而/但/吊诡"检测）                          │
    │     • 是否没给结论？（禁用"因此/总之"检测）                         │
    │     • 是否没AI泄漏？（套话列表检测）                                  │
    │                                                                             │
    │  ③ 反思结果分析：                                                          │
    │     • 全部通过 → 进入Step 6                                          │
    │     • 未通过且第1次 → 生成修正提示，调用LLM重生成（复用已有概念）         │
    │     • 未通过且第2次 → 追加免责声明，返回最佳结果（不再重试）              │
    │                                                                             │
    │  关键保证：最多2次迭代，绝不无限循环。                                   │
    └───────────────────────────────────────────────────────────────┘
                                        │
    Step 6: O6 人性化处理（3区扫描）              │
    ┌───────────────────────────────────────────────────────────────┐
    │  Zone 1: AI泄漏区 → 删除"让我们/首先/值得注意的是/从另一个角度"        │
    │  Zone 2: 温度过高区 → 降低"非常/极其/深刻的/重要的"等过热词汇         │
    │  Zone 3: 开了玖风味保护区 → 检测第一人称/鸡汤/补义说教类语言            │
    │                                                                             │
    │  发现风味违规时标记HUMANIZER_FLAG，供调用方判断是否需要重生成。       │
    └───────────────────────────────────────────────────────────────┘
